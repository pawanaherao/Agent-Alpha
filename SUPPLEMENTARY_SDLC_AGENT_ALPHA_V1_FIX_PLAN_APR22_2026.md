## 1. Mandate

This document is the supplementary SDLC submission for fixing and hardening Agent Alpha V1.

It is not a V2 migration note.

It converts the attached V1 audit findings into an implementation program with:

- completed fixes already delivered
- remaining fix scope by severity and workstream
- expert team ownership
- phase-wise implementation sequence
- validation gates required before broader paper-trading or live-shadow expansion

## 2. Executive Position

Agent Alpha V1 will be retained and improved in place.

The audit established that V1 already contains usable foundations:

- FastAPI, Redis, PostgreSQL, scheduler, and Docker runtime base
- working orchestration cycle skeleton in AgentManager
- real manual controls with SEBI audit persistence
- real Day Commander and Cycle Intelligence concepts
- real diagnostics engine
- real GenAI validation influence in StrategyAgent

The failure mode of V1 is not total absence of architecture. The failure mode is fragmented authority across control, AI, and orchestration boundaries.

Therefore the SDLC objective is:

1. keep the working V1 runtime base
2. remove control-plane drift and dead paths
3. unify runtime authority before adding more intelligence
4. validate by executable checks, not documents alone
